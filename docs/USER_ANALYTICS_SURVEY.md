**Purpose**: Gather user preferences for Spotify analytics dashboard features
**Target Audience**: Potential users, students, music enthusiasts

## Google Form Outline

## Implementation Notes

### Google Forms Setup

1. **Create form**: https://forms.google.com
2. **Settings**:
   - ✅ Collect email addresses (for beta testers)
   - ✅ Limit to 1 response
   - ✅ Show progress bar
   - ✅ Shuffle question order (for some sections to reduce bias)

3. **Response Collection**:
   - Link to Google Sheets for analysis
   - Use Google Forms built-in analytics for quick insights
   - Export to CSV for deeper analysis in Python/pandas

### Question Mapping to Available Data

**What We CAN Deliver** (from DATA_ARCHITECTURE_RECOMMENDATION_REVISED.md):
- ✅ Top artists/tracks for 3 time ranges (short/medium/long term)
- ✅ Recently played 50 tracks with timestamps
- ✅ Temporal patterns (hour, day, week from timestamps)
- ✅ Genre analysis (from artist data)
- ✅ Popularity scores (mainstream vs niche)
- ✅ Diversity metrics (unique artists/tracks ratio)
- ✅ Taste consistency (overlap between time ranges)
- ✅ Artist follower counts
- ✅ Explicit content ratio
- ✅ Release year distributions
- ✅ Historical trends (requires multiple snapshots)

**What We CANNOT Deliver** (due to API limitations):
- ❌ Direct audio features (danceability, energy, valence, tempo, mood)
  - Workaround: Kaggle dataset lookup (~60-80% coverage)
  - Survey questions acknowledge this limitation
- ❌ Play counts per track (API doesn't provide)
  - Workaround: Rankings and "recently played" only
- ❌ Real-time live updates
  - Limitation: 24-hour refresh cycle recommended

### Analysis Plan

After collecting responses:

1. **Priority Matrix**: Plot "interest level" vs "implementation effort"
2. **Feature Voting**: Count which features got most selections
3. **User Segments**: Cluster users by preferences
   - Casual users (just want overview)
   - Data enthusiasts (want all details)
   - Social sharers (want shareable graphics)
   - Explorers (want historical trends)

4. **Dashboard Structure Decision**:
   - Map high-priority features to dashboard pages
   - Determine single-page vs multi-page layout
   - Prioritize first-time user experience vs returning user features
